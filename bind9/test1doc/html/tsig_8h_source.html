107 /*%<
108  * Creates a tsig key structure and saves it in the keyring. If key is
109  * not NULL, *key will contain a copy of the key. The keys validity
110  * period is specified by (inception, expire), and will not expire if
111  * inception == expire. If the key was generated, the creating identity,
112  * if there is one, should be in the creator parameter. Specifying an
113  * unimplemented algorithm will cause failure only if dstkey != NULL; this
114  * allows a transient key with an invalid algorithm to exist long enough
115  * to generate a BADKEY response.
116  *
117  * If dns_tsigkey_createfromkey is successful a new reference to 'dstkey'
118  * will have been made.
119  *
120  * Requires:
121  *\li 'name' is a valid dns_name_t
122  *\li 'algorithm' is a valid dns_name_t
123  *\li 'secret' is a valid pointer
124  *\li 'length' is an integer >= 0
125  *\li 'dstkey' is a valid dst key or NULL
126  *\li 'creator' points to a valid dns_name_t or is NULL
127  *\li 'mctx' is a valid memory context
128  *\li 'ring' is a valid TSIG keyring or NULL
129  *\li 'key' or '*key' must be NULL
130  *
131  * Returns:
132  *\li #ISC_R_SUCCESS
133  *\li #ISC_R_EXISTS - a key with this name already exists
134  *\li #ISC_R_NOTIMPLEMENTED - algorithm is not implemented
135  *\li #ISC_R_NOMEMORY
136  */
