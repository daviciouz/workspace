98 /*%
99  ***** An 'rdata' is a handle to a binary region. The handle has an RR
100  ***** class and type, and the data in the binary region is in the format
101  ***** of the given class and type.
102  *****/
103 /*%
104  * Clients are strongly discouraged from using this type directly, with
105  * the exception of the 'link' field which may be used directly for whatever
106  * purpose the client desires.
107  */

115 };

438 /*%<
439  * Like dns_rdata_totext, but do formatted output suitable for
440  * database dumps. This is intended for use by dns_db_dump();
441  * library users are discouraged from calling it directly.
442  *
443  * If (flags & #DNS_STYLEFLAG_MULTILINE) != 0, attempt to stay
444  * within 'width' by breaking the text into multiple lines.
445  * The string 'linebreak' is inserted between lines, and parentheses
446  * are added when necessary. Because RRs contain unbreakable elements
447  * such as domain names whose length is variable, unpredictable, and
448  * potentially large, there is no guarantee that the lines will
449  * not exceed 'width' anyway.
450  *
451  * If (flags & #DNS_STYLEFLAG_MULTILINE) == 0, the rdata is always
452  * printed as a single line, and no parentheses are used.
453  * The 'width' and 'linebreak' arguments are ignored.
454  *
455  * If (flags & #DNS_STYLEFLAG_COMMENT) != 0, output explanatory
456  * comments next to things like the SOA timer fields. Some
457  * comments (e.g., the SOA ones) are only printed when multiline
458  * output is selected.
459  *
460  * base64 rdata text (e.g., DNSKEY records) will be split into chunks
461  * of 'split_width' characters. If split_width == 0, the text will
462  * not be split at all. If split_width == UINT_MAX (0xffffffff), then
463  * it is undefined and falls back to the default value of 'width'
464  */
