397 /*%<
398  * Specify the parameters of a logging channel.
399  *
400  * Notes:
401  *\li The name argument is copied to memory in the logging context, so
402  * it can be altered or destroyed after isc_log_createchannel().
403  *
404  *\li Defining a very large number of channels will have a performance
405  * impact on isc_log_usechannel(), since the names are searched
406  * linearly until a match is made. This same issue does not affect
407  * isc_log_write, however.
408  *
409  *\li Channel names can be redefined; this is primarily useful for programs
410  * that want their own definition of default_syslog, default_debug
411  * and default_stderr.
412  *
413  *\li Any channel that is redefined will not affect logging that was
414  * already directed to its original definition, _except_ for the
415  * default_stderr channel. This case is handled specially so that
416  * the default logging category can be changed by redefining
417  * default_stderr. (XXXDCL Though now that I think of it, the default
418  * logging category can be changed with only one additional function
419  * call by defining a new channel and then calling isc_log_usechannel()
420  * for #ISC_LOGCATEGORY_DEFAULT.)
421  *
422  *\li Specifying #ISC_LOG_PRINTTIME or #ISC_LOG_PRINTTAG for syslog is
423  * allowed, but probably not what you wanted to do.
424  *
425  * #ISC_LOG_DEBUGONLY will mark the channel as usable only when the
426  * debug level of the logging context (see isc_log_setdebuglevel)
427  * is non-zero.
428  *
429  * Requires:
430  *\li lcfg is a valid logging configuration.
431  *
432  *\li name is not NULL.
433  *
434  *\li type is #ISC_LOG_TOSYSLOG, #ISC_LOG_TOFILE, #ISC_LOG_TOFILEDESC or
435  * #ISC_LOG_TONULL.
436  *
437  *\li destination is not NULL unless type is #ISC_LOG_TONULL.
438  *
439  *\li level is >= #ISC_LOG_CRITICAL (the most negative logging level).
440  *
441  *\li flags does not include any bits aside from the ISC_LOG_PRINT* bits,
442  * #ISC_LOG_DEBUGONLY or #ISC_LOG_BUFFERED.
443  *
444  * Ensures:
445  *\li #ISC_R_SUCCESS
446  * A channel with the given name is usable with
447  * isc_log_usechannel().
448  *
449  *\li #ISC_R_NOMEMORY or #ISC_R_UNEXPECTED
450  * No additional memory is being used by the logging context.
451  * Any channel that previously existed with the given name
452  * is not redefined.
453  *
454  * Returns:
455  *\li #ISC_R_SUCCESS Success
456  *\li #ISC_R_NOMEMORY Resource limit: Out of memory
457  *\li #ISC_R_UNEXPECTED type was out of range and REQUIRE()
458  * was disabled.
459  */

465 /*%<
466  * Associate a named logging channel with a category and module that
467  * will use it.
468  *
469  * Notes:
470  *\li The name is searched for linearly in the set of known channel names
471  * until a match is found. (Note the performance impact of a very large
472  * number of named channels.) When multiple channels of the same
473  * name are defined, the most recent definition is found.
474  *
475  *\li Specifying a very large number of channels for a category will have
476  * a moderate impact on performance in isc_log_write(), as each
477  * call looks up the category for the start of a linked list, which
478  * it follows all the way to the end to find matching modules. The
479  * test for matching modules is integral, though.
480  *
481  *\li If category is NULL, then the channel is associated with the indicated
482  * module for all known categories (including the "default" category).
483  *
484  *\li If module is NULL, then the channel is associated with every module
485  * that uses that category.
486  *
487  *\li Passing both category and module as NULL would make every log message
488  * use the indicated channel.
489  *
490  * \li Specifying a channel that is #ISC_LOG_TONULL for a category/module pair
491  * has no effect on any other channels associated with that pair,
492  * regardless of ordering. Thus you cannot use it to "mask out" one
493  * category/module pair when you have specified some other channel that
494  * is also used by that category/module pair.
495  *
496  * Requires:
497  *\li lcfg is a valid logging configuration.
498  *
499  *\li category is NULL or has an id that is in the range of known ids.
500  *
501  * module is NULL or has an id that is in the range of known ids.
502  *
503  * Ensures:
504  *\li #ISC_R_SUCCESS
505  * The channel will be used by the indicated category/module
506  * arguments.
507  *
508  *\li #ISC_R_NOMEMORY
509  * If assignment for a specific category has been requested,
510  * the channel has not been associated with the indicated
511  * category/module arguments and no additional memory is
512  * used by the logging context.
513  * If assignment for all categories has been requested
514  * then _some_ may have succeeded (starting with category
515  * "default" and progressing through the order of categories
516  * passed to isc_log_registercategories()) and additional memory
517  * is being used by whatever assignments succeeded.
518  *
519  * Returns:
520  *\li #ISC_R_SUCCESS Success
521  *\li #ISC_R_NOMEMORY Resource limit: Out of memory
522  */
