149 /*%<
150  * Method prototype. Drivers implementing the SDLZ interface MUST
151  * supply a find zone method. This method is called when the DNS
152  * server is performing a query to to determine if 'name' is a
153  * supported dns zone. The find zone method will be called with the
154  * longest possible name first, and continue to be called with
155  * successively shorter domain names, until any of the following
156  * occur:
157  *
158  * \li 1) the function returns (ISC_R_SUCCESS) indicating a zone name
159  * match.
160  *
161  * \li 2) a problem occurs, and the functions returns anything other than
162  * (ISC_R_NOTFOUND)
163  *
164  * \li 3) we run out of domain name labels. I.E. we have tried the
165  * shortest domain name
166  *
167  * \li 4) the number of labels in the domain name is less than min_labels
168  * for dns_dlzfindzone
169  *
170  * The driver's find zone method should return ISC_R_SUCCESS if the
171  * zone is supported by the database. Otherwise it should return
172  * ISC_R_NOTFOUND, if the zone is not supported. If an error occurs
173  * it should return a result code indicating the type of error.
174  */
