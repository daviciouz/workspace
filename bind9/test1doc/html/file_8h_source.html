362 /*%<
363  * Generate a sanitized filename, such as for MKEYS or NZF files.
364  *
365  * Historically, MKEYS and NZF files used SHA256 hashes of the view
366  * name for the filename; this was to deal with the possibility of
367  * forbidden characters such as "/" being in a view name, and to
368  * avoid problems with case-insensitive file systems.
369  *
370  * Given a basename 'base' and an extension 'ext', this function checks
371  * for the existence of file using the old-style name format in directory
372  * 'dir'. If found, it returns the path to that file. If there is no
373  * file already in place, a new pathname is generated; if the basename
374  * contains any excluded characters, then a truncated SHA256 hash is
375  * used, otherwise the basename is used. The path name is copied
376  * into 'path', which must point to a buffer of at least 'length'
377  * bytes.
378  *
379  * Requires:
380  * - base != NULL
381  * - path != NULL
382  *
383  * Returns:
384  * - ISC_R_SUCCESS on success
385  * - ISC_R_NOSPACE if the resulting path would be longer than 'length'
386  */
