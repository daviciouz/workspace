42  * When some atomic instructions with hardware assistance are
43  * available, rwlock will use those so that concurrent readers do not
44  * interfere with each other through mutex as long as no writers
45  * appear, massively reducing the lock overhead in the typical case.

47  * The basic algorithm of this approach is the "simple
48  * writer-preference lock" shown in the following URL:
49  * http://www.cs.rochester.edu/u/scott/synchronization/pseudocode/rw.html
50  * but our implementation does not rely on the spin lock unlike the
51  * original algorithm to be more portable as a user space application.
52  */
