218 /*%<
219  * General, low-level ACL matching. This is expected to
220  * be useful even for weird stuff like the topology and sortlist statements.
221  *
222  * Match the address 'reqaddr', and optionally the key name 'reqsigner',
223  * and optionally the client prefix 'ecs' of length 'ecslen'
224  * (reported via EDNS client subnet option) against 'acl'.
225  *
226  * 'reqsigner' and 'ecs' may be NULL. If an ACL matches against 'ecs'
227  * and 'ecslen', then 'scope' will be set to indicate the netmask that
228  * matched.
229  *
230  * If there is a match, '*match' will be set to an integer whose absolute
231  * value corresponds to the order in which the matching value was inserted
232  * into the ACL. For a positive match, this value will be positive; for a
233  * negative match, it will be negative.
234  *
235  * If there is no match, *match will be set to zero.
236  *
237  * If there is a match in the element list (either positive or negative)
238  * and 'matchelt' is non-NULL, *matchelt will be pointed to the matching
239  * element.
240  *
241  * 'env' points to the current ACL environment, including the
242  * current values of localhost and localnets and (if applicable)
243  * the GeoIP context.
244  *
245  * Returns:
246  *\li #ISC_R_SUCCESS Always succeeds.
247  */
