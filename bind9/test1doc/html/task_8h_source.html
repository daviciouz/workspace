708  * Set/get the current operating mode of the task manager. Valid modes are:
709  *
710  *\li isc_taskmgrmode_normal

712  *
713  * In privileged execution mode, only tasks that have had the "privilege"
714  * flag set via isc_task_setprivilege() can be executed. When all such
715  * tasks are complete, the manager automatically returns to normal mode
716  * and proceeds with running non-privileged ready tasks. This means it is
717  * necessary to have at least one privileged task waiting on the ready
718  * queue *before* setting the manager into privileged execution mode,
719  * which in turn means the task which calls this function should be in
720  * task-exclusive mode when it does so.

729 /*%<
730  * Destroy '*managerp'.
731  *
732  * Notes:
733  *
734  *\li Calling isc_taskmgr_destroy() will shutdown all tasks managed by
735  * *managerp that haven't already been shutdown. The call will block
736  * until all tasks have entered the done state.
737  *
738  *\li isc_taskmgr_destroy() must not be called by a task event action,
739  * because it would block forever waiting for the event action to
740  * complete. An event action that wants to cause task manager shutdown
741  * should request some non-event action thread of execution to do the
742  * shutdown, e.g. by signaling a condition variable or using
743  * isc_app_shutdown().
744  *
745  *\li Task manager references are not reference counted, so the caller
