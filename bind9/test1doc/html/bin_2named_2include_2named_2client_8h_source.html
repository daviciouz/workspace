310 /*%
311  * Convenience function for client request ACL checking.
312  *
313  * Check the current client request against 'acl'. If 'acl'
314  * is NULL, allow the request iff 'default_allow' is ISC_TRUE.
315  * If netaddr is NULL, check the ACL against client->peeraddr;
316  * otherwise check it against netaddr.
317  *
318  * Notes:
319  *\li This is appropriate for checking allow-update,
320  * allow-query, allow-transfer, etc. It is not appropriate
321  * for checking the blackhole list because we treat positive
322  * matches as "allow" and negative matches as "deny"; in
323  * the case of the blackhole list this would be backwards.
324  *
325  * Requires:
326  *\li 'client' points to a valid client.
327  *\li 'netaddr' points to a valid address, or is NULL.
328  *\li 'acl' points to a valid ACL, or is NULL.
329  *
330  * Returns:
331  *\li ISC_R_SUCCESS if the request should be allowed
332  * \li DNS_R_REFUSED if the request should be denied
333  *\li No other return values are possible.
334  */
