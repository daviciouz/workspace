101 /*%<
102  * Register a simple database driver for the database type 'drivername',
103  * implemented by the functions in '*methods'.
104  *
105  * sdbimp must point to a NULL dns_sdbimplementation_t pointer. That is,
106  * sdbimp != NULL && *sdbimp == NULL. It will be assigned a value that
107  * will later be used to identify the driver when deregistering it.
108  *
109  * The name server will perform lookups in the database by calling the
110  * function 'lookup', passing it a printable zone name 'zone', a printable
111  * domain name 'name', and a copy of the argument 'dbdata' that
112  * was potentially returned by the create function. The 'dns_sdblookup_t'
113  * argument to 'lookup' and 'authority' is an opaque pointer to be passed to
114  * ns_sdb_putrr().
115  *
116  * The lookup function returns the lookup results to the name server
117  * by calling ns_sdb_putrr() once for each record found. On success,
118  * the return value of the lookup function should be ISC_R_SUCCESS.
119  * If the domain name 'name' does not exist, the lookup function should
120  * ISC_R_NOTFOUND. Any other return value is treated as an error.
121  *
122  * Lookups at the zone apex will cause the server to also call the
123  * function 'authority' (if non-NULL), which must provide an SOA record
124  * and NS records for the zone by calling ns_sdb_putrr() once for each of
125  * these records. The 'authority' function may be NULL if invoking
126  * the 'lookup' function on the zone apex will return SOA and NS records.
127  *
128  * The allnodes function, if non-NULL, fills in an opaque structure to be
129  * used by a database iterator. This allows the zone to be transferred.
130  * This may use a considerable amount of memory for large zones, and the
131  * zone transfer may not be fully RFC1035 compliant if the zone is
132  * frequently changed.
133  *
134  * The create function will be called for each zone configured
135  * into the name server using this database type. It can be used
136  * to create a "database object" containing zone specific data,
137  * which can make use of the database arguments specified in the
138  * name server configuration.
139  *
140  * The destroy function will be called to free the database object
141  * when its zone is destroyed.
142  *
143  * The create and destroy functions may be NULL.
144  *
145  * If flags includes DNS_SDBFLAG_RELATIVEOWNER, the lookup and authority
146  * functions will be called with relative names rather than absolute names.
147  * The string "@" represents the zone apex in this case.
148  *
149  * If flags includes DNS_SDBFLAG_RELATIVERDATA, the rdata strings may
150  * include relative names. Otherwise, all names in the rdata string must
151  * be absolute. Be aware that if relative names are allowed, any
152  * absolute names must contain a trailing dot.
153  *
154  * If flags includes DNS_SDBFLAG_THREADSAFE, the driver must be able to
155  * handle multiple lookups in parallel. Otherwise, calls into the driver
156  * are serialized.
157  */
