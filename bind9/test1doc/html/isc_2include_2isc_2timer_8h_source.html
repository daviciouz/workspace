170 /*%<
171  * Create a new 'type' timer managed by 'manager'. The timers parameters
172  * are specified by 'expires' and 'interval'. Events will be posted to
173  * 'task' and when dispatched 'action' will be called with 'arg' as the
174  * arg value. The new timer is returned in 'timerp'.
175  *
176  * Notes:
177  *
178  *\li For ticker timers, the timer will generate a 'tick' event every
179  * 'interval' seconds. The value of 'expires' is ignored.
180  *
181  *\li For once timers, 'expires' specifies the time when a life timeout
182  * event should be generated. If 'expires' is 0 (the epoch), then no life
183  * timeout will be generated. 'interval' specifies how long the timer
184  * can be idle before it generates an idle timeout. If 0, then no
185  * idle timeout will be generated.
186  *
187  *\li If 'expires' is NULL, the epoch will be used.
188  *
189  * If 'interval' is NULL, the zero interval will be used.
190  *
191  * Requires:
192  *
193  *\li 'manager' is a valid manager
194  *
195  *\li 'task' is a valid task
196  *
197  *\li 'action' is a valid action
198  *
199  *\li 'expires' points to a valid time, or is NULL.
200  *
201  *\li 'interval' points to a valid interval, or is NULL.
202  *
203  *\li type == isc_timertype_inactive ||
204  * ('expires' and 'interval' are not both 0)
205  *
206  *\li 'timerp' is a valid pointer, and *timerp == NULL
207  *
208  * Ensures:
209  *
210  *\li '*timerp' is attached to the newly created timer
211  *
212  *\li The timer is attached to the task
213  *
214  *\li An idle timeout will not be generated until at least Now + the
215  * timer's interval if 'timer' is a once timer with a non-zero
216  * interval.
217  *
218  * Returns:
219  *
220  *\li Success
221  *\li No memory
222  *\li Unexpected error
223  */
224
