682 /*%<
683  * Copy the possibly-compressed name at source (active region) into target,
684  * decompressing it.
685  *
686  * Notes:
687  * \li Decompression policy is controlled by 'dctx'.
688  *
689  * \li If DNS_NAME_DOWNCASE is set, any uppercase letters in 'source' will be
690  * downcased when they are copied into 'target'.
691  *
692  * Security:
693  *
694  * \li *** WARNING ***
695  *
696  * \li This routine will often be used when 'source' contains raw network
697  * data. A programming error in this routine could result in a denial
698  * of service, or in the hijacking of the server.
699  *
700  * Requires:
701  *
702  * \li 'name' is a valid name.
703  *
704  * \li 'source' is a valid buffer and the first byte of the active
705  * region should be the first byte of a DNS wire format domain name.
706  *
707  * \li 'target' is a valid buffer or 'target' is NULL and 'name' has
708  * a dedicated buffer.
709  *
710  * \li 'dctx' is a valid decompression context.
711  *
712  * Ensures:
713  *
714  * If result is success:
715  * \li If 'target' is not NULL, 'name' is attached to it.
716  *
717  * \li Uppercase letters are downcased in the copy iff
718  * DNS_NAME_DOWNCASE is set in options.
719  *
720  * \li The current location in source is advanced, and the used space
721  * in target is updated.
722  *
723  * Result:
