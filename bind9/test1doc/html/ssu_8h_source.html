130  * to the rules specified in the simple-secure-update rule table. If
131  * no rules are matched, access is denied.
132  *
133  * Notes:
134  * 'tcpaddr' should only be set if the request received
135  * via TCP. This provides a weak assurance that the
136  * request was not spoofed. 'tcpaddr' is to to validate
137  * DNS_SSUMATCHTYPE_TCPSELF and DNS_SSUMATCHTYPE_6TO4SELF
138  * rules.
139  *
140  * For DNS_SSUMATCHTYPE_TCPSELF the addresses are mapped to
141  * the standard reverse names under IN-ADDR.ARPA and IP6.ARPA.
142  * RFC 1035, Section 3.5, "IN-ADDR.ARPA domain" and RFC 3596,
143  * Section 2.5, "IP6.ARPA Domain".
144  *
145  * For DNS_SSUMATCHTYPE_6TO4SELF, IPv4 address are converted
146  * to a 6to4 prefix (48 bits) per the rules in RFC 3056. Only
147  * the top 48 bits of the IPv6 address are mapped to the reverse
148  * name. This is independent of whether the most significant 16
149  * bits match 2002::/16, assigned for 6to4 prefixes, or not.
